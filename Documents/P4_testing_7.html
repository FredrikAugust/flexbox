Feedback summary and response

Most of the feedback was very good. Users generally appreciated the overall professional look of the website. A particular feature users seemed to enjoy was the hijack-scrolling and the expanding menus. A few users made positive comments regarding the use of strong and specific colors for each season. One user said the website made them "hungry".

We received some useful feedback regarding bugs on mobile platforms. One particular issue on mobile platforms was scrolling. After receiving feedback we added a media query for mobile devices and turned disabled hijack-scrolling for these devices. There was also a minor issue with responsiveness on the table reservation form. This was fixed after testing.

One issue some users seemed to have was that they didn't understand they were supposed to scroll down from the front page. We intended to have a scroll indicator, represented by dots at the right side of the page, with the dot of the current page being highlighted. However, we encountered some problems with this and decided not to include it. This, or some other, scroll indicator would probably make it more obvious for the users that they were supposed to scroll.

We had forgotten to fill out the wine menus before starting testing. This was changed by a hotfix while tests were being conducted.

Another minor issue was that the text on the front page was a bit difficult to read on some monitor resolutions because the text appeared right in front of some of the very bright leaves. After testing we added a slightly transparent panel behind the text. If we were to work on this project further, we could maybe have been done better by setting how the background image would scale on different resolutions.

Some minor testing was done after these fixes were made. This feedback was only conducted via live testing, and no feedback was submitted by a new Google Form. This feedback was only positive, with no bugs or other negative feedback reported.

Self-reflection

All of us have past experiences in web design and/or other programming, and expected to do quite well on this project. We have mostly accomplished what we intended, but we realized that working as a team is very different from working alone. Some of us had experience with git and version control, but not all. This resulted in a few merge conflicts and a lot of additional, unexpected work. Although we did plan ahead, quite comprehensively in fact, we also encountered some problems with the development itself. These issues were mostly related to javascript, which some of us had little or no past experience with. In hindsight, we may have been slightly too ambitious when planning the features of the website.
