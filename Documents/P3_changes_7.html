P3 project changes

Authors

Fredrik August Madsen-Malmo, Erik Skår, Andreas Winther Moen, Leonard Nguyen Schøyen

Date

11/11-2018

In general, the end result was fairly similar to the intended website specified in P2. We ran into a few problems, particularly related to some of the Javascript features. Details are specified below and in P4.

Changes to front page

The original plan was to have a jumbotron video, replaced by a still image on mobile platforms. We attempted to record a video of the forest which would then be sped up, but due to lack of good enough equipment, in particular a tripod, the video didn't turn out very well. The video was both shaky and the brightness varied too much throughout the video. We tried to edit it slightly, but none of us had past experience with video production. Also, since this is not a required skill for the project, we chose to use a still image on all platforms instead.

Changes to scrolling

As specified in P4, hijack-scrolling turned out to be a bit more difficult than anticipated. We managed to complete the feature, but it didn't work that well on mobile, so we made mobile devices scroll like standard. We also had to remove the visual scroll indicator presented in P2. More information about this is provided in P4.

Changes to season menus

First we set the color of the season menu panels to be the average of the background images. This turned out to be a bit difficult ro read, and generally didn't look well with the background. Instead we set the panels to be a more distinctive color which also made it easy to read. We also added a border with a different color to set it better apart from the background.

Changes to font

As specified in P2, the font was not fully determined. The font from P2, Poiret One, was very modernistic. However, as we started developing the website we realized we needed something more reminiscent of Norwegian nature, and changed it to Playfair Display. The name of the restaurant uses the font Amatic SC instead. We gave the name a special font to make it stand out.

Changes to script.js

As we started working separately on specific features, we realized it would be better to add separate Javascript files for each feature. This would make the code more readable and organized, and also prevent merge conflicts. Because of this, script.js was split into formfeedback.js (table reservation form), lazy.js (lazy loading of images), menu.js (visual expansion and contraction of menus upon hovering), scroll.js (hijack-scrolling) and wine.js (wine menu). This proved to be quite useful.

Changes to logo

While a logo was made, it didn't fit the theme of the website. Neither of us are visual artists, and the logo looked more cartoon-like than something that would fit a restaurant targeted towards upper-class consumers. Instead, we made a simplified version of the logo and set it as the favicon of the website.
